🗄 1. Configurar Supabase

O banco de dados central. Gerencia autenticação, dados das empresas e isolamento por cliente (RLS).

1
Criar conta em supabase.com
Login com GitHub é o mais rápido. A conta gratuita suporta até 2 projetos ativos.
2
New project → configurações
Nome: produzai · Região: South America (São Paulo) · Senha: gere uma forte e guarde em local seguro.
3
Executar o SQL de setup
Painel → SQL Editor → New query → cole o conteúdo completo de supabase_setup.sql → clique em Run.

O script cria: tabelas empresas, usuarios, pessoas, registros, planos · índices de performance · políticas RLS · funções RPC · triggers de updated_at.
4
Copiar as credenciais: Settings → API
Você precisa de dois valores:
• Project URL: https://xxxxxxxx.supabase.co
• anon public key: começa com eyJhbGci...
5
Colar no app: Config → Supabase → Conectar
O status fica verde ao conectar. O app migra automaticamente do localStorage para o banco.

🤖 2. API Key Anthropic

Usada para o reconhecimento de formulários por foto (Claude Vision). Cada empresa pode ter sua própria chave — ou você pode gerenciar centralmente.

1
Criar conta em console.anthropic.com
Conta Business ou Team recomendada para uso comercial. O plano Free tem limites de uso.
2
API Keys → Create Key
Nome: produzai-producao. Copie a chave gerada (começa com sk-ant-api03-). Ela não fica visível novamente depois.
3
Definir onde a chave fica
Opção A (SaaS): Cada empresa cadastra sua própria chave em Config → API Key. Cada uma paga diretamente pela Anthropic.

Opção B (Gerenciado): Você mantém uma chave central e embutida no código. Mais simples, mas você absorve os custos de reconhecimento.

▲ 3. Deploy no Vercel

Hospedagem gratuita com CDN global e SSL automático. O webhook do Stripe roda como Serverless Function.

1
Criar conta em vercel.com
Login com GitHub. O plano Hobby (gratuito) é suficiente para começar.
2
Add New → Project → Browse → selecionar a pasta
Arraste ou selecione a pasta com todos os arquivos do projeto. O Vercel detecta automaticamente como site estático.
3
Adicionar o webhook como API Route
Renomeie stripe_webhook.js para api/stripe-webhook.js antes do upload. O Vercel reconhece automaticamente como função serverless.
4
Configurar variáveis de ambiente
Painel Vercel → Settings → Environment Variables. Adicione as variáveis abaixo.
5
Domínio personalizado (opcional)
Settings → Domains → adicione app.seudominio.com.br. O Vercel provisiona o SSL automaticamente via Let's Encrypt.

Variáveis de ambiente no Vercel

STRIPE_SECRET_KEY
sk_live_... (painel Stripe → Developers → API Keys)
STRIPE_WEBHOOK_SECRET
whsec_... (gerado ao criar o webhook no Stripe)
SUPABASE_URL
https://xxxxxxxx.supabase.co
SUPABASE_SERVICE_KEY
eyJ... (Settings → API → service_role key — NÃO a anon key)
🚨
Atenção: O webhook usa a service_role key (não a anon). Essa chave tem acesso total ao banco — nunca exponha no frontend.

💳 4. Configurar Stripe

Gerencia as assinaturas mensais. Os planos são controlados pelo painel Stripe e refletidos automaticamente no banco via webhook.

1
Criar conta em stripe.com
Preencha os dados da sua empresa para habilitar cobranças em produção. Em modo de teste, use as chaves sk_test_.
2
Criar os produtos e preços
Products → Add product:
• ProduzAI Básico → R$99/mês (recorrente mensal)
• ProduzAI Profissional → R$199/mês (recorrente mensal)
Copie os price_id gerados (formato: price_1ABC...).
3
Atualizar o mapa de preços em stripe_webhook.js
Abra o arquivo e preencha o objeto PRICE_TO_PLANO: 
const PRICE_TO_PLANO = {
  'price_SEU_ID_BASICO_MENSAL':  'basico',
  'price_SEU_ID_PRO_MENSAL':     'pro',
};
4
Configurar o webhook no Stripe
Developers → Webhooks → Add endpoint
URL: https://seudominio.vercel.app/api/stripe-webhook
Eventos a escutar: customer.subscription.created, customer.subscription.updated, customer.subscription.deleted, checkout.session.completed, invoice.payment_failed
5
Atualizar a tabela planos no Supabase
Supabase → Table Editor → planos → edite as linhas:
• Preencha stripe_price_id_mensal com o price_id real
• Os botões de assinatura no app ficam ativos automaticamente

💰 Gerenciar planos

Para alterar preços ou features dos planos, você tem dois pontos a atualizar:

Alterar o preço

1
No Stripe: criar um novo price
Não edite o preço existente — o Stripe não permite. Vá em Products → seu produto → Add another price → defina o novo valor.
2
No Supabase: atualizar a tabela planos
Table Editor → planos → edite a linha correspondente: atualize preco_mensal, stripe_price_id_mensal com o novo price_id.
3
No stripe_webhook.js: atualizar o mapa
Adicione o novo price_id no objeto PRICE_TO_PLANO e faça novo deploy no Vercel.

Alterar features exibidas na tela de preços

No Supabase → Table Editor → planos → coluna features. É um array JSON. Exemplo:

Supabase — tabela planos → coluna features
["Até 5 usuários", "Dashboard completo", "Relatórios PDF", "Suporte por email"]

Salve e o app exibe automaticamente as novas features na próxima vez que a tela de preços for aberta.

👥 Gerenciar clientes

Ver todas as empresas cadastradas

Supabase → Table Editor → tabela empresas. Mostra todas as empresas com plano, data de trial, status da assinatura e stripe_customer_id.

Estender o trial de um cliente

Supabase → SQL Editor → execute:

SQL — estender trial
UPDATE empresas
SET trial_expira = now() + interval '7 days'
WHERE email = 'cliente@empresa.com';

Bloquear/desbloquear acesso

SQL — bloquear empresa
UPDATE empresas
SET assinatura_ativa = false
WHERE email = 'cliente@empresa.com';

🔍 Troubleshooting

App não conecta ao Supabase

Verifique se a URL e a anon key estão corretas em Config → Supabase. A URL deve começar com https:// e terminar com .supabase.co. Confirme que o SQL de setup foi executado sem erros.

Reconhecimento por foto não funciona

Confirme que a API Key está salva em Config → API Key Anthropic. Verifique no console do navegador (F12) se há erro de CORS ou de autenticação. A chave deve ter créditos disponíveis na conta Anthropic.

Webhook do Stripe não ativa o plano

No painel Stripe → Developers → Webhooks → seu endpoint → verifique os eventos recentes. Erros 401 indicam problema com STRIPE_WEBHOOK_SECRET. Erros 500 indicam problema no código do webhook — verifique os logs no Vercel (Functions → Logs).

Cliente cadastrado mas não aparece como empresa no Supabase

A função RPC criar_empresa_e_admin pode ter falhado. No Supabase → Authentication → Users, veja se o usuário foi criado. Se sim, execute manualmente:

SQL — criar empresa manualmente
SELECT criar_empresa_e_admin(
  'Nome da Empresa',
  'email@empresa.com',
  'Nome do Usuário'
);
